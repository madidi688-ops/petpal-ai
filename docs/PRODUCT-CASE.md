# PetPal AI · 产品案例（求职 / 作业作答）

> 对应作答要求：机会判断 · 用户证据 · MVP 样例 · 商业判断 · 2 周验证计划  
> 并补充：**模型分工与降级决策**（AI 产品岗高频追问点）  
> 版本：2026-07-16 · 作品：可运行 MVP（Next.js + NestJS + DeepSeek / 火山方舟）

---

## 0. 一句话定位

**PetPal AI** 是给「把宠物当家人的年轻铲屎官」用的 **情绪陪伴 + 长期记忆** 产品：用多模态对话让宠物「会回应」，用行为记录驱动 AI 日记，把碎片日常沉淀成可回顾、可分享的陪伴资产——而不是又一个万能宠物 GPT。

---

## 1. 机会判断

### 目标用户是谁？

**核心用户（ICP）**

- 20–35 岁，一二线城市，独居或伴侣不常在家的猫/狗主
- 把宠物当作家人 /「另一个自己」，在小红书等社区高频分享养宠日常
- 愿意为情绪价值付费（冻干、玩具、智能硬件周边已有消费习惯），但 **尚未有稳定的「数字陪伴」习惯**

**非目标（刻意不做）**

- 兽医诊疗决策工具、训宠指令大全、电商导购机器人  
- 纯虚拟宠物养成（无真实宠物绑定）——本产品叙事绑定真实毛孩子

### 痛点是什么？

| 痛点 | 表现 | 现有替代的不足 |
|---|---|---|
| **单向陪伴** | 下班回家只有叫声与肢体接触，想「被懂」却无法对话 | 朋友圈自言自语；宠物翻译器常被吐槽「随机猜谜」 |
| **日常易碎** | 吃/玩/睡的瞬间记不住，想发日记没素材 | 备忘录零散；相册堆着不回顾 |
| **情绪无处安放** | 独居压力大，宠物是情绪出口但无法「接住」语言层需求 | 通用 ChatGPT 人设漂移、像客服、无宠物记忆 |

### 为什么现在值得做？

1. **多模态能力拐点**：图/短视频/语音理解成本下降，使「发一段猫吃饭视频 → 它用第一人称回应」从噱头变成可演示体验。  
2. **养宠情绪消费升级**：公开行业洞察持续强调「宠物=家人 / 另一个自己」、科学养宠与情绪价值并存（见下节证据）。  
3. **竞品缝隙清晰**：翻译器偏「猜」、通用 Bot 无长期行为记忆；PetPal 卡在 **陪伴人格 × 行为沉淀 × 日记资产** 的交叉点。  
4. **个人可验证**：纯软件 MVP 两周内可上路演；硬件绑定不是冷启动必需。

## 2. 用户证据

> 诚实边界：本节以 **公开行业报告 + 社区常见表述归纳** 为主；一手访谈放在「2 周验证计划」补齐。求职答辩时应主动说明证据层级。

### 2.1 公开数据 / 报告（二手）

- 小红书 × CBNData《2024 宠物行业洞察》：平台养宠内容与决策高度集中；趋势强调科学养宠、专宠专用与「双向奔赴」式人宠关系。  
  参考：[CBNData 报道摘要](https://www.cbndata.com/information/292504)
- 社区侧描述：小红书养宠泛兴趣与实际养宠用户规模进入「亿 / 千万」量级讨论；用户常把宠物当作家人甚至「另一个自己」。  
  参考：[胖萌舍转述峰会观点](https://www.pmshe.com/86953.html)
- 情绪消费语境：独居与「空巢青年」场景下，宠物深度介入情绪生活；异宠等内容高互动，本质是陪伴需求细分。  
  参考：[CBNData 异宠与情绪消费](https://m.cbndata.com/information/295382)
- 人群结构：白皮书口径下 90 后等年轻宠主占比高，决策与分享场仍在内容社区（广告侧洞察亦同向）。  
  参考：[小红书宠物人群洞察相关转载](https://m.xiao-ad.com/al/1789.html)

### 2.2 社区常见原声类型（归纳，非单条私采）

| 类型 | 典型表述（归纳） | 对产品含义 |
|---|---|---|
| 想对话 | 「好想知道它在想什么」「翻译器不准但还是想玩」 | 要人格体验，容忍不完美，但不能像鉴定报告 |
| 想记录 | 「想写养宠日记但好懒」「相册一堆不知道发啥」 | 行为一记 + 一键日记降低创作成本 |
| 想被陪 | 「一个人住，只有猫等我」 | 晚间短互动 > 长功能列表 |
| 不信任 | 「又是 AI 装宠物」「不要给我乱开药」 | 人设约束 + 医疗安全拒答进 eval |

### 2.3 证据缺口（主动暴露）

- 缺少 ≥8 场结构化用户访谈与付费意愿卡片排序  
- 缺少「日记分享」真实转发率数据  
→ 由第 5 节 2 周计划补齐，避免用报告替代验证。

## 3. MVP 样例

### 3.1 核心闭环（用户可感知）

```text
登录演示号 → 发猫短视频聊天 → 记 1 条行为 → 生成今日日记（分享卡可截图）
```

账号：`demo@petpal.ai` / `demo1234`（seed：年糕带头像、样例会话、行为、日记）

### 3.2 可演示资产

| 资产 | 位置 |
|---|---|
| 可运行 Web | 本地 `localhost:3000`（见 README 快速启动） |
| 演示脚本 / 录屏分镜 | [`docs/DEMO.md`](./DEMO.md) |
| Prompt 与人设 | `backend/src/ai/prompts/chat-pet.ts`（`chat-pet@v2`） |
| 离线 Eval | `npm run eval` → `ai-agents/evals/` |
| 本案例页 | 本文 |

### 3.3 AI 工作流（简化）

```text
用户消息 (±图/视/音)
    ├─ 纯文本 ──────────► DeepSeek Chat
    └─ 含媒体 ──────────► 火山方舟 Responses API
                              (图/视: ARK_MODEL；音: ARK_AUDIO_MODEL，wav)
         │
         ▼
   人设 System Prompt（第一人称宠物，禁自称 AI）
         │
         ▼
   回复流式 SSE + EmotionLog
         │
行为事件 ──► 日记生成 / MBTI 刷新（独立链路，可降级文案）
```

### 3.4 模型与降级决策（产品决策页核心）

| 场景 | 选型 | 为什么 | 失败时 |
|---|---|---|---|
| 纯文本陪伴 | DeepSeek | 成本低、中文闲聊稳、够用 | Mock / 友好中文「稍后再试」 |
| 图 / 短视频理解 | 方舟多模态接入点 | DeepSeek 不支持图；方舟 Responses 可演示「正在看视频」 | 提示稍后再试；不装懂 |
| 语音 | 方舟音频接入点 + 前端转 wav | 浏览器 webm 不被接受 | 同上；HTTPS 才稳定开麦 |
| 无 Key / 演示 | 演示模式 | 保证路演与求职官 clone 可跑 | 仍可看 UI 与 seed 内容 |
| 人设 / 安全 | Prompt + Eval | 防止「我是 AI」「乱开药」 | Eval 红灯阻断「自以为好」 |

**刻意不做的决策**：不做「一个模型打天下」；不做强医疗建议；不做首发就绑定硬件。

## 4. 商业判断

### 付费用户是谁？

- **个人订阅**：重度陪伴用户（每周多日对话 + 日记）  
- **潜在 B 端**（后期）：宠物品牌 / 门店的「养宠社群运营工具」（非 MVP 焦点）

### 为什么付费？

付费买的不是「多一轮 Chat」，而是：

1. **稳定人设的长期记忆**（行为 → 日记连续性）  
2. **多模态额度**（视频理解贵，适合作为订阅墙）  
3. **分享资产**（日记卡、纪念日回顾）带来的社交货币  

免费层：文本陪伴 + 有限日记；付费层：视频理解次数、历史回溯、多宠物。

### 如何定价与获客（假设，待验证）

| 项 | 初案 | 验证方式 |
|---|---|---|
| 定价 | ¥12–25 / 月，或 ¥68 / 季；视频理解按次附加 | 访谈 van Westendorp；落地页 A/B |
| 获客 | 小红书「养宠日记」内容种草 → 落地页 → Demo | 2 周发 9 条内容看咨询 / 注册 |
| 转化钩子 | 生成一张好看日记分享卡 | 追踪「生成日记 → 截图分享」自报 |

**单位经济粗算（示意）**：若月订阅 ¥18，毛利需覆盖 DeepSeek + 方舟多模态；故 **免费用户限视频次数**，避免被一条 50MB 视频打穿成本。

## 5. 两周验证计划

> 目标：用最小资源回答「有没有人愿意持续用 / 愿意为多模态或日记付费意向」。

### Week 1 — 证据与漏斗

| 天 | 动作 | 产出 |
|---|---|---|
| 1–2 | 8 场 25 分钟访谈（猫主 5 / 狗主 3），卡片排序：翻译器 vs 日记 vs 视频聊 | 痛点优先级表 |
| 3–4 | 落地页 + 预约 Demo；小红书发 3 条「发视频它会回」 | 预约名单 |
| 5–7 | 10 人远程可用性：固定 2 分钟路径；录屏问题 | 完成率、卡点列表 |

**资源**：本人全职；访谈礼品 ¥50×8；域名 + 简易托管 ≈ ¥100；模型 API 预算 ¥200。

### Week 2 — 付费意向与质量

| 天 | 动作 | 产出 |
|---|---|---|
| 8–9 | 对完成日记的用户做付费意向（假门：订阅按钮点了但未扣款） | 点击率 |
| 10–11 | 补真模型 eval 抽样 20 条（人设 / 医疗拒答） | pass rate 报告 |
| 12–14 | 复盘：留 / 改 / 砍；输出下个两周只做 1 个北极星 | 决策备忘 |

**北极星假说（先写死再打）**：  
「7 日内生成 ≥3 篇日记的用户，次周回访率显著更高」——两周内用事件日志近似验证，不追求完美埋点。

**成功标准（预声明）**

- 访谈中 ≥5/8 将「视频回应」或「一键日记」排进前二  
- 可用性完成率 ≥70% 走完演示路径  
- 假门订阅点击 ≥15%（小样本仅作方向）  

未达则：**砍 MBTI 展示优先级**，集中「视频聊 + 日记卡」。

## 6. 求职官可追问的准备句

- **为什么不做翻译器？** 用户要的是情绪接住，不是词对齐；我们优化人设与记忆。  
- **幻觉怎么办？** Prompt 禁 AI 自称 + 离线 eval；医疗场景拒处方。  
- **为什么双模型？** 能力边界与成本；产品上要让用户感到「看懂了」且账单可控。  
- **两周后若失败？** 承认情绪不成立，沉淀工作流与 eval，不沉没在功能堆砌。
